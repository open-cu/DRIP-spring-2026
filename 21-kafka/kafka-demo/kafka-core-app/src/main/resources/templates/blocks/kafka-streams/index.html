Блок kafka-streams: topology внутри kafka-core-app

Учебный сценарий: берем событие заказа, пропускаем его через topology (схему обработки потока) и смотрим результат в новом topic.

Это позднее расширение kafka-core-app, а не отдельный demo-модуль.

Вернуться в Kafka core

Сценарий

Обычный listener принимает сообщение и вызывает сервис. Здесь мы описываем маленький поток: берем событие из input topic (входного канала), преобразуем его и кладем результат в result topic (канал результата).

В демо вы отправляете описание заказа. Kafka Streams читает строку из orders.stream.input, добавляет признак STREAM_ACCEPTED, приводит описание к верхнему регистру и пишет новое сообщение в orders.stream.result.

  • Listener удобен, когда нужна точка входа в бизнес-метод.
  • Kafka Streams удобен, когда вы хотите описать цепочку обработки как topology (схему потока).
  • В этой лекции мы показываем идею, без тяжелого разбора state store (хранилища состояния) и window (окна по времени).

Путь события

1. Input topic

orders.stream.input принимает учебный заказ.

2. Topology

OrderStreamsTopology преобразует payload (тело сообщения).

3. Result topic

orders.stream.result хранит результат, который читает UI.
