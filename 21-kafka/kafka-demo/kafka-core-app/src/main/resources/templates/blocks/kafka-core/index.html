Блок kafka-core: первый рабочий путь события

Учебный сценарий: создаем заказ, отправляем событие в topic и смотрим, как listener запускает обработку.

Сценарий

Клиент создает заказ. Spring-код отправляет событие OrderCreated в topic (канал событий), а listener (метод-читатель) получает его и запускает обработку заказа.

Сначала смотрим самый короткий путь: producer (отправитель) -> topic -> consumer (получатель).

  • KafkaTemplate: место, где приложение явно публикует событие.
  • @KafkaListener: обычный Spring-метод, который Kafka вызывает после чтения сообщения.
  • Если статус обновился не мгновенно, это нормально: событие сначала проходит через broker.
