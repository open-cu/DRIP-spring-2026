Блок kafka-distribution-model: key, partition, consumer group

Учебный сценарий: отправляем события в topic и смотрим, какую partition выбрала Kafka.

Одинаковый key помогает сохранить порядок событий одного заказа внутри одной partition.

Сценарий

Отправьте несколько событий с одним и тем же key (ключ события), например order-42. Kafka выберет для них одну partition (дорожку потока), и таблица покажет partition, offset (номер сообщения) и consumer (получатель), который прочитал событие.

Так видно, почему порядок событий одного заказа обычно привязывают к key, а параллельность чтения упирается в число partition.

  • Порядок Kafka держит внутри одной partition, а не во всем topic сразу.
  • Если partition три, четвертый consumer в той же group (группе читателей) не ускорит чтение.
  • При rebalance (перераспределении partition) обработчик должен спокойно переживать повторы.
