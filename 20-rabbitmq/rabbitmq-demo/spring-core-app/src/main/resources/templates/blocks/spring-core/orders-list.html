Блок spring-core: бизнес-поток “заказ -> обработка -> статус”

Это учебная модель обработки заказа в бэкенде интернет-магазина: API создаёт заказ, сообщение уходит в RabbitMQ, consumer обрабатывает, затем можно запросить статус через RPC.

На странице видно 4 учебных сценария Spring AMQP: очередь заказов, отправка строки с exchange, отправка только сообщения (default exchange), RPC запрос статуса.

Поток обработки (как в реальном сервисе)

  1. Создание: менеджер/клиент создаёт заказ через HTTP форму.
  2. Публикация: producer отправляет событие заказа в RabbitMQ.
  3. Обработка: consumer читает очередь и меняет статус на “ОБРАБОТАН”.
  4. Проверка: при необходимости вызываем RPC и получаем актуальный статус.

Действия для демонстрации

1) Новый заказ

Базовый бизнес-кейс: заказ попадает в очередь, потом обрабатывается consumer-ом.

2) Отправка строки (выбор режима)

Один endpoint, два учебных режима отправки: явный convertAndSend(exchange, routingKey, message) или convertAndSend(message).

EXPLICIT: exchange
EXPLICIT: routingKey
DEFAULT: routingKey из rabbitTemplate defaults
